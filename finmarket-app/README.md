# FinMarket TJ — MVP (демо)

Финансовый маркетплейс банковских продуктов Таджикистана. Полный рабочий MVP: каталог, фильтры, сравнение, онлайн-заявка, маршрутизация лидов, админ-панель с RBAC и аудитом.

Реализован как **единое Next.js-приложение** (фронт + API + админка в одном), на **SQLite** и **in-memory очереди** — чтобы запускаться без Docker, PostgreSQL и RabbitMQ. Нужен только Node.js.

## Что нужно установить (один раз)

- **Node.js 20+** — https://nodejs.org (LTS). Больше ничего.

Проверка: `node -v` (должно быть v20 или выше).

## Запуск (3 команды)

```bash
npm install          # установка зависимостей
npm run setup        # генерация Prisma + создание БД + наполнение демо-данными
npm run dev          # запуск приложения
```

Открыть в браузере: **http://localhost:3000**

> При первом `npm run setup` Prisma скачает движки (~40 МБ) — нужен интернет один раз. Дальше всё работает офлайн.

## Полезные команды

| Команда | Что делает |
|---|---|
| `npm run dev` | запуск в режиме разработки (localhost:3000) |

## Как посмотреть весь сценарий

1. **Главная** → выбери категорию «Кредиты».
2. **Листинг** → примени фильтр по ставке, добавь 2-3 продукта «+ К сравнению».
3. **Сравнение** (в шапке) → увидишь подсветку лучшего значения (★).
4. **Карточка продукта** → полные условия, комиссии, требования.
5. **Подать заявку** → заполни форму (телефон в формате РТ), отметь согласие → экран успеха.
6. **Админ** → войди как admin → вкладка **Заявки**: увидишь свою заявку, статус сменится `new → sent` (доставка лида) автоматически.
7. **Продукты** → опубликуй/сними черновик; **Аудит** → запись об изменении с diff.

## Что реализовано из ТЗ

| Сравнение (до 4, одна категория, подсветка) | FR-CMP-01..05, US-03, BR-02 |

| Lead Routing (webhook+HMAC / email, retry, DLQ) | FR-LR-01..05, US-06, BR-05/06 |
| Admin (CRUD, лиды, resend, аудит, RBAC) | FR-ADM-01..07, US-07/08, BR-01/07 |
| Локализация tj/ru (переключатель) | US-09 |

## Архитектура (демо vs production)

| Аспект | В демо | В production (по ТЗ) |
|---|---|---|
| БД | SQLite (файл) | PostgreSQL 15 |
| Очередь лидов | in-memory | RabbitMQ + DLQ |
| Auth | демо-токен | Keycloak (OIDC) |
| Структура | единый Next.js | монорепо (web/admin/api) |

Переключение на production-стек делается заменой `datasource` в `prisma/schema.prisma` и реализацией очереди — бизнес-логика и API-контракты не меняются.

```
│   │   ├── apply/[productId]/       # форма заявки
│   │   ├── admin/                   # админ-панель (SPA)
│   │   └── api/                     # все REST-эндпоинты
│   ├── components/Header.tsx
│   └── lib/
│       ├── db.ts                    # Prisma client
│       ├── validation.ts            # телефон РТ, ошибки, атрибуты сравнения
│       ├── lead-router.ts           # очередь + retry + DLQ + HMAC
│       ├── auth.ts                  # RBAC + аудит
│       └── store.ts                 # сравнение + i18n (Zustand)
└── docs/                            # ТЗ и приложения (OpenAPI, схемы, Gherkin)
```
